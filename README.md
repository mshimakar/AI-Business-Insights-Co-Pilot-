## üß† How the AI Works (The Engine)

The power of the Co-Pilot comes from its specialized architecture. 

The system translates your request through the following steps:

1.  **Intent Classification & Context Retrieval:** The system determines the query's goal and simultaneously retrieves relevant business rules from the vector store.
2.  **SQL Agent Execution:** The **LangChain SQL Agent** dynamically generates and executes the most efficient **PostgreSQL** query against the live data.
3.  **Analytics Layer:** Raw results are passed to the built-in **Pandas** engine for advanced computations, such as **Forecasting** (predicting future trends) or **Anomaly Detection** (identifying statistical outliers).
4.  **Insight Synthesis:** The analytical results, along with the original question and relevant business rules, are packaged and sent to the **Large Language Model (LLM)**, which generates the final, easily understandable insight presented to you.

## ‚ùì Frequently Asked Questions (FAQ)

### Q: What data can the Co-Pilot access?
A: It is currently configured to analyze data from the **Sales**, **Customer Support Tickets**, and **User** databases.

### Q: Can I ask for a forecast?
A: Yes. You can ask, "Forecast next quarter's sales revenue," or "What is the expected ticket volume next week?" The system uses a dedicated time-series model (e.g., Prophet) to generate predictions.

### Q: What if I get an error?
A: If the AI cannot generate a valid SQL query or finds no matching data, it will return a message stating the limitations. Please check your spelling and try rephrasing the question to be more specific (e.g., specify a date range).
